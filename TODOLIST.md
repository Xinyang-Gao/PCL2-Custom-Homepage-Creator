## 🔥 新发现的问题（待修复）

以下问题在本次代码审查中新识别出，尚未列入上述优先级，建议尽快处理：

| 编号 | 问题 | 描述 | 影响 | 建议修复 |
| ---- | ---- | ---- | ---- | ---- |
| #F1  | `App.recordSnapshot` 未定义 | `uiManager.js` 和 `xamlProcessor.js` 中调用了 `App.recordSnapshot()`，但 `appCore.js` 未实现该方法。 | 清除画布、导入 XAML 时会抛出 `TypeError`，功能中断。 | 删除所有 `App.recordSnapshot()` 调用，或实现一个空方法（历史记录已由增量操作管理，不再需要快照）。 |
| #F2  | Margin 滑块实时调整未记录历史 | `renderManager.js` 中 `updateComponentMargin` 直接修改属性并刷新样式，但不触发历史记录。 | 用户调整边距后无法撤销/重做。 | 在滑块 `mouseup` 事件中调用 `applyCurrentProps` 统一提交变更；或记录单独的 `UPDATE_PROPS` 操作。 |
| #F3  | 历史操作中移动组件时索引可能漂移 | 撤销移动操作时依赖记录的 `oldIndex`，但若移动后其他组件增减，索引可能无效。 | 理论上撤销可能错位，但因每次移动后立即记录，实际发生概率低。 | 在 `doMove` 中根据组件 ID 重新查找当前位置，而不是依赖索引。 |
| #F4  | Grid 布局未实际渲染行列 | 生成的 XAML 包含列/行定义，但预览时 Grid 容器仅显示为普通块级元素，子元素未按网格定位。 | 网格布局的视觉效果缺失，用户无法调整子元素位置。 | 需要为 Grid 容器生成 CSS Grid 样式（`display: grid; grid-template-columns/rows`），并根据子组件的附加属性放置。 |
