## 🟠 P1 – 高优先级（功能缺陷或重大体验）

| 编号 | 问题 | 说明 | 解决方案 |
|------|------|------|----------|
| **#A1** | Grid 列/行定义导出丢失部分属性 | 虽然导入时存储了 `minWidth`/`maxWidth`，但 `Grid.ColumnDefinition` 还支持 `Width` 含 `Auto`、`*` 等，且导出时未正确处理 `Width="Auto"` 的输出格式（可能输出 `"Auto"` 字串正确，但需检查数值型和星号格式）。 | 在 `generateXAML` 中，对于列/行定义，确保 `Width`/`Height` 属性按原样输出，并保留 `MinWidth`、`MaxWidth`、`MinHeight`、`MaxHeight`。 |
| **#A2** | 事件绑定功能缺失 | 编辑器内可设置 `EventType` 和 `EventData`，但预览时按钮无实际响应（如打开网页、弹窗等）。 | 在 `RenderManager` 中为可交互组件（按钮、文本按钮、列表项）添加 `click` 事件监听，根据 `EventType` 执行对应操作（`window.open`、`alert`、复制文本等）。 |
| **#A3** | 导入 XAML 时组件 ID 可能冲突 | `importFromXAML` 中使用 `getNextLocalId()` 重新生成 ID，但未考虑与现有组件 ID 的全局唯一性（`getMaxGlobalId` 只读当前树，而新导入时使用独立递增）。 | 在 `parseNode` 中调用 `ComponentManager.getNextId()` 而不是局部计数器，确保全局唯一。 |
| **#A4** | 撤销/重做性能问题（全量快照） | 每次变动都深拷贝整个 `components` 树，组件数量大时（>200）会导致明显卡顿。 | 改为增量状态存储（操作日志）或使用结构共享（如 Immer），或限制快照频率（仅关键操作记录）。 |

## 🟡 P2 – 中优先级（体验优化与增强）

| 编号 | 任务 | 说明 | 解决方案 |
|------|------|------|----------|
| **#B1** | 快捷键全面支持 | 缺失 Ctrl+S（保存到本地文件）、Delete（删除选中组件）、Ctrl+D（复制组件）。 | 在 `window` 键盘事件中增加对应分支，调用 `fileManager.saveToLinkedFile`、`ComponentManager.removeComponentById`、`ComponentManager.duplicateComponent`。 |
| **#B2** | 拖拽时滚动跟随 | 拖拽组件接近画布边缘时，预览区不能自动滚动。 | 在 `dragover` 中检测鼠标位置与容器边界的距离，动态调整 `scrollTop`/`scrollLeft`。 |
| **#B3** | 组件库拖拽预览图 | 从左侧拖拽组件时无跟随鼠标的占位符。 | 使用 `setDragImage` 自定义拖拽图像，显示组件图标和名称。 |
| **#B4** | 备份管理增强 | 备份列表无搜索、无批量删除；恢复前无差异对比。 | 增加搜索过滤框；支持多选删除；可选的“差异对比”弹窗。 |
| **#B5** | 超大组件树初次渲染优化 | 目前仍采用全量 `innerHTML` 重建，组件 > 200 时卡顿。 | 引入虚拟滚动（仅渲染可视区域）或使用增量 DOM 更新（保留现有节点，仅增删改移动）。 |
| **#B6** | 网格布局编辑器增强 | 当前仅支持简单添加/删除，不支持拖拽调整顺序、复制列/行定义。 | 在网格编辑器中增加排序按钮（上移/下移）和复制定义功能。 |
| **#B7** | 属性面板实时响应时防抖 | 输入框 `change` 事件立即触发 `applyCurrentProps` 并刷新组件，高频输入可能导致重复渲染。 | 对 `applyCurrentProps` 增加防抖（如 150ms），仅在停止输入后应用。 |
| **#B8** | 导入 XAML 时保留未知元素为注释 | 未知标签直接被忽略，用户可能丢失内容。 | 将未知元素序列化为 XML 注释保留在组件树的特殊字段中，导出时还原。 |

## 🔵 P3 – 低优先级（性能、代码质量、边缘情况）

| 编号 | 任务 | 说明 | 解决方案 |
|------|------|------|----------|
| **#C1** | 图片组件 `EnableCache` 等属性未暴露 | 属性面板中缺少 `EnableCache`、`FallbackSource`、`LoadingSource`。 | 在 `ComponentTypes.image.defaults` 中补充，并在属性面板通用分组中显示。 |
| **#C2** | 列表项 `Logo` 属性不支持 SVG 路径或内置图标预览 | 目前仅显示默认图标，无法预览用户自定义的 Logo。 | 在 `RenderManager` 的 `listitem` 渲染中，尝试将 `Logo` 作为 `<img>` 显示，或使用 FontAwesome 图标映射。 |
| **#C3** | 代码分割与模块化 | 所有逻辑仍堆积在 `script.js`，未拆分为独立文件。 | 将 `HistoryManager`、`ComponentManager`、`RenderManager`、`XamlProcessor`、`FileManager`、`ServerApi`、`DragDropManager`、`UIManager` 分别放入独立模块，使用 ES6 导入/导出。 |
| **#C4** | 移动端触屏拖拽 | 手机/平板上无法拖拽组件。 | 使用 Touch 事件模拟 drag & drop，或增加“编辑模式”开关，通过点击添加组件。 |
| **#C5** | 多语言支持 | 界面全中文，不利于国际化。 | 提取所有 UI 文本到语言包（`locales/zh.js`、`en.js`），根据浏览器或用户选择加载。 |
| **#C6** | 导出 XAML 时的格式选项 | 当前缩进固定 2 空格，用户可能希望自定义缩进或移除注释。 | 在 XAML 预览弹窗中增加格式设置（缩进大小、是否保留空行等）。 |
| **#C7** | 分享设计为 URL | 可将当前设计编码为 URL 参数，便于分享。 | 将组件树 JSON 压缩并 base64 放入 URL hash，页面加载时自动恢复设计（需注意长度限制）。 |
