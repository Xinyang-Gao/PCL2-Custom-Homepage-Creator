### 🟡 仍需关注的问题

| 类别 | 问题 | 优先级 | 说明 |
|------|------|--------|------|
| 健壮性 | 大文件分块读取对小文件不友好 | P3 | 即使几 KB 的文件也分块读取（1MB 块），增加不必要的异步开销。可增加判断：若文件 < 1MB 则直接一次性读取。 |
| 健壮性 | 拖拽添加组件时的顺序丢失 | P3 | `appendComponentToParent` 总是将新组件追加到父容器的末尾，而非原插入位置附近（例如拖拽到现有组件之间）。因设计未要求精确排序，可接受；若后续需要，可扩展。 |
| 用户体验 | 容器删除后子组件的选中状态残留 | P3 | 删除一个容器组件后，其子组件的数据已移除，但若子组件之前被选中，`selectedId` 被置为 `null`，不会出现错误。但无额外提示。可优化为选中父组件的父级。 |
| 功能 | 未知组件类型导入时静默跳过 | P2 | XAML 中包含非支持的组件标签（如自定义第三方控件）会被忽略，无警告提示，可能导致用户数据丢失。应在解析时收集未知标签并弹窗警告。 |
| 性能 | 超大组件树（>200）初次渲染仍用全量 | P3 | `renderCanvas` 仍是全量构建 DOM，虽然增删改使用了增量，但首次加载或清空后重建仍会一次渲染所有组件。可考虑虚拟滚动，但当前场景罕见。 |
| 健壮性 | 组件 `nextId` 校准时机不完整 | P2 | `App.init()` 中调用 `getMaxGlobalId()` 校准，但如果在运行时通过外部脚本直接修改 `App.state.components` 而不经过 `importFromXAML`（如控制台操作），`nextId` 可能小于现有 ID。但正常使用无问题。 |
