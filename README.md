## 🟠 P1 – 高优先级（功能缺陷或重大体验问题）

| 编号 | 问题 | 说明 | 解决方案 |
|------|------|------|----------|
| **#1** | 水平布局拖拽插入位置错误 | 在 `Orientation="Horizontal"` 的 `StackPanel` 中拖拽组件时，仍根据鼠标 Y 坐标计算插入索引，导致顺序错乱。 | 在 `computeDropPosition` 中根据父容器的 `isHorizontal` 动态选择 `clientX` 或 `clientY`；同时调整占位符样式和位置计算逻辑。 |
| **#2** | Grid 列/行定义解析不完整 | 导入 XAML 时仅读取 `ColumnDefinition` 的 `Width` 属性，丢失 `MinWidth`、`MaxWidth` 等；导出时也仅输出宽度。 | 修改 `importFromXAML` 中 Grid 定义解析，保留所有属性；导出时原样输出。建议将列/行定义存储为结构化对象而非简单字符串。 |
| **#3** | 通用属性编辑缺失 | `ToolTip`、`IsHitTestVisible` 等通用属性未在属性面板固定显示，用户无法直观修改。 | 在属性面板增加「通用」分组，始终显示常用通用属性（Margin, ToolTip, HorizontalAlignment, VerticalAlignment, IsHitTestVisible）。 |
| **#4** | 卡片特有属性未暴露 | `MyCard` 的 `UseAnimation`、`SwapLogoRight`、`HasMouseAnimation` 等属性在编辑器中不可编辑。 | 在 `ComponentTypes.card.defaults` 中补充这些属性，并在属性面板中提供对应输入控件（布尔类型用下拉或复选框）。 |
| **#5** | 重复组件插入位置不当 | 复制组件时直接追加到父容器末尾，而非原组件之后，破坏布局顺序。 | 修改 `duplicateComponent`，在原组件的下一个位置插入克隆，并调用增量 DOM 插入。 |
| **#6** | 导入 XAML 时事件属性未正确处理 | `EventType` 和 `EventData` 会被读取并存入 `comp.events`，但若节点上同时存在其他自定义属性，可能被覆盖。 | 代码已基本正确，但需确保导入时不会错误地覆盖已有属性。测试边界情况（如同时存在 `EventType` 和同名自定义属性）。 |

## 🟡 P2 – 中优先级（体验优化与重要功能增强）

| 编号 | 问题 | 说明 | 解决方案 |
|------|------|------|----------|
| **#7** | 自定义属性管理 | 用户通过手动编辑 XAML 添加的非标准属性（如特殊布局属性）被保存在 `customProps` 中，但无法在属性面板中编辑或删除。 | 在属性面板增加“自定义属性”区域，支持动态增删改键值对，并实时同步到 `comp.customProps`。 |
| **#8** | 撤销 / 重做功能 | 目前无历史状态栈，误操作无法回退。 | 实现历史管理器（最多 50 步），监听所有组件变更（增删改移动），保存状态快照。快捷键 Ctrl+Z / Ctrl+Y。 |
| **#9** | 网格布局可视化编辑 | 列/行定义仅支持文本输入（如 `1*,2*,Auto`），无法直观调整比例、最小宽度等。 | 为 Grid 组件增加可视化编辑器弹窗：展示列/行列表，支持拖拽调整大小、设置单位（像素/星/自动）、最小/最大宽度等。 |
| **#10** | 预览样式与真实 PCL 差异 | 编辑器内按钮、卡片等样式与 PCL 实际运行效果差距较大（颜色、圆角、悬停效果等）。 | 参考 PCL 的 WPF 主题，调整 CSS 变量和按钮样式，尽量接近真实表现（至少颜色、间距一致）。 |
| **#11** | 属性面板实时响应 | 修改属性后需点击“应用更改”才生效，不够流畅。 | 监听所有输入控件的 `input`/`change` 事件，实时更新组件 props 并刷新预览（对性能敏感属性可防抖）。 |
| **#12** | 组件库搜索高亮 | 搜索组件时匹配项无视觉反馈。 | 在搜索结果中高亮匹配的文字，或增加匹配计数。 |
| **#13** | 备份管理增强 | 备份列表无搜索、无批量删除；恢复备份前无差异对比。 | 增加备份搜索框；支持多选删除；可选的“与当前设计对比”功能（显示差异）。 |
| **#14** | 快捷键支持 | 缺少常用快捷键（保存、撤销、删除、复制等）。 | 监听全局键盘事件：Ctrl+S（保存到本地文件）、Ctrl+D（复制选中组件）、Delete（删除）、Ctrl+Z/Y（撤销/重做）。 |
| **#15** | 拖拽时滚动跟随 | 拖拽组件到画布边缘时，预览区不能自动滚动。 | 在 `dragover` 中检测鼠标位置距离边缘阈值，自动滚动容器。 |

## 🔵 P3 – 低优先级（性能、代码质量、边缘情况）

| 编号 | 问题 | 说明 | 解决方案 |
|------|------|------|----------|
| **#16** | 超大组件树全量渲染卡顿 | 组件数量 > 200 时，每次修改都全量重建 DOM（尽管已部分优化，仍有回退情况）。 | 全面采用增量渲染：对 `renderCanvas` 进行重构，改为基于虚拟 DOM 的 diff 更新，或确保所有操作（添加、删除、移动、属性更改）都走增量路径，绝不回退全量渲染。 |
| **#17** | Margin 输入框非数字容错 | 输入非数字时 `parseFloat` 返回 `NaN`，导致最终 XAML 出现 `NaN,NaN,NaN,NaN`。 | 在 `applyCurrentProps` 中对 margin 各值校验，无效值默认 0；输入框失焦时重置为有效数值。 |
| **#18** | 属性面板频繁重建 DOM | 每次选中组件都销毁并重建所有属性分组，浪费性能。 | 改为增量更新：仅修改变化字段的值，而非重建整个面板。或使用 `requestIdleCallback` 已做优化，但可进一步减少重绘。 |
| **#19** | 导入 XAML 时无进度提示（大文件） | 虽然实现了分块读取，但进度弹窗样式简陋。 | 美化进度弹窗，增加取消按钮，支持用户中断导入。 |
| **#20** | 组件库拖拽时无预览图 | 从左侧拖拽组件时无跟随鼠标的占位符。 | 使用 `setDragImage` 自定义拖拽图像，显示组件图标和名称。 |
| **#21** | 代码分割与模块化 | 所有逻辑堆积在 `script.js`，未来维护困难。 | 将 `RenderManager`、`ComponentManager`、`DragDropManager` 等拆分为独立文件，使用 ES6 模块。 |

## ⚪ P4 – 可选 / 锦上添花

| 编号 | 问题 | 说明 | 解决方案 |
|------|------|------|----------|
| **#22** | 移动端触屏拖拽 | 在手机/平板上无法拖拽组件。 | 使用 Touch 事件模拟 drag & drop，或增加“编辑模式”开关，通过点击添加组件。 |
| **#23** | 多语言支持 | 界面全中文，不利于国际化。 | 提取所有 UI 文本到语言包（如 `locales/zh.js`、`en.js`），根据浏览器或用户选择加载。 |
| **#24** | 组件库分组和收藏 | 组件过多时查找不便。 | 将组件分为“基础控件”“布局容器”“交互组件”等分组，支持折叠/展开；允许用户收藏常用组件到置顶区。 |
| **#25** | 导出 XAML 时的格式选项 | 当前固定缩进 2 空格，用户可能希望自定义缩进或移除注释。 | 在 XAML 预览弹窗中增加格式设置（缩进大小、是否保留空行等）。 |
| **#26** | 分享设计为 URL | 可将当前设计编码为 URL 参数，便于分享。 | 将组件树 JSON 压缩并 base64 放入 URL hash，页面加载时自动恢复设计。 |
