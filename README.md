## 🟠 功能缺陷（仍需修复）

### 1. XAML 导入解析不完整（Grid 附加属性 + 命名空间前缀）
- **文件**：`script.js` → `XamlProcessor.importFromXAML`
- **现状**：未解析子元素上的 `Grid.Row` / `Grid.Column` 等附加属性；命名空间前缀匹配仅靠 `tagName.includes('mycard')`，脆弱。
- **修复方案**：
  - 解析时读取 `node.getAttribute('Grid.Row')`、`Grid.Column`，存入 `comp.props`。
  - 在生成 XAML 时为子元素输出 `Grid.Row="..."` 属性。
  - 支持 `local:MyCard`、`mycard` 等多种写法，使用正则 `/^(?:local:)?mycard$/i`。
- **优先级**：🟠 P1

### 2. 本地文件“保存到链接文件”对传统 `input` 打开的文件无效
- **文件**：`script.js` → `FileManager.saveToLinkedFile`
- **现状**：当浏览器不支持 File System Access API 或用户通过传统 `<input>` 打开文件时，`currentFileHandle === null`，保存按钮被禁用或提示不支持，用户只能“另存为”。
- **改进方案**：
  - 在传统方式打开文件后，将文件内容存为“临时工作区”，并提供“另存为覆盖”的流程。
  - 若用户再次点击“保存到链接文件”，自动调用 `saveAsLocalFile` 并更新 `currentFileName`。
- **优先级**：🟠 P1

### 3. 拖拽大文件无加载反馈
- **文件**：`DragDropManager.handleFileImport`
- **现状**：直接 `FileReader.readAsText` 阻塞 UI，无进度提示或遮罩。
- **修复方案**：
  - 显示模态 loading（“正在解析大文件…”）。
  - 若文件 > 5MB，提示可能耗时。
  - 读取完成后关闭 loading。
- **优先级**：🟠 P1

### 4. 服务器备份列表焦点恢复不完善
- **文件**：`UIManager.renderBackupList` 及 `ServerApi`
- **现状**：删除备份后刷新列表，但未保留之前选中的备份项（如有），也不记忆上次加载的备份。
- **修复方案**：
  - 在 `ServerApi` 中记录 `currentSelectedBackup`。
  - 删除或恢复操作后，重新渲染时尝试选中该备份（若还存在），否则选中第一个或显示“无选中”。
- **优先级**：🟠 P1

## 🟡 健壮性与性能

### 5. XAML 解析错误定位不友好
- **文件**：`XamlProcessor.importFromXAML`
- **现状**：错误信息仅截取前 100 字符，无行列号。
- **修复方案**：
  - 从 `parsererror` 元素中提取行号/列号（部分浏览器支持）。
  - 利用 `DOMParser` 的 `error.lineNumber`（需遍历节点）。
  - 显示为：`第 12 行第 8 列：标签未闭合`。
- **优先级**：🟡 P2

### 6. 后端并发写入无文件锁
- **文件**：`app.py` 中的 `/api/backup`、`/api/save`、`/api/local/save`
- **现状**：多请求同时写同一文件可能交错损坏内容。
- **修复方案**：
  - 写入临时文件（`filepath.tmp`）后原子替换 `os.replace`。
  - 或使用 `portalocker` 或 `fcntl.flock`。
- **优先级**：🟡 P2

### 7. 属性面板切换组件时渲染卡顿（同步渲染）
- **文件**：`RenderManager.updatePropsPanel`
- **现状**：复杂组件树（>50 组件）切换选中时属性面板同步构建 DOM，有短暂空白。
- **修复方案**：
  - 使用 `requestAnimationFrame` 分批渲染属性分组。
  - 当属性超过 30 个时，虚拟滚动或分页。
- **优先级**：🟡 P2

### 8. 组件复制时全局 `nextId` 未基于当前最大 ID 动态计算
- **文件**：`ComponentManager.deepCloneComponent`
- **现状**：`App.state.nextId` 只在初始化和导入时更新，连续复制多个组件时不会冲突，但若手动设置过组件 ID（未来可能）或外部导入 ID 跳跃，则可能重复。
- **修复方案**：
  - 在 `deepCloneComponent` 开头调用 `App.state.nextId = getMaxGlobalId() + 1`。
  - 确保每次克隆前 ID 基准确。
- **优先级**：🟡 P2

## 🔵 用户体验

### 9. 关闭页面无未保存提示
- **文件**：`script.js` 初始化
- **现状**：用户可关闭页面丢失未保存更改。
- **修复方案**：
  - 监听 `beforeunload`，若 `App.state.dirty === true`，返回标准提示字符串。
  - 注意仅在真正有未保存内容时阻止。
- **优先级**：🔵 P3

### 10. 部分按钮状态反馈不够直观
- **文件**：`UIManager.updateOpenFileButton`
- **现状**：保存文件后按钮短暂变为“已是最新”后又恢复为“打开本地文件”，可能让用户困惑。
- **改进方案**：
  - 增加 toast 提示“已保存”即可，不需要改变按钮文字回退效果。
  - 或者持续显示“文件已同步”标记，直到下一次修改。
- **优先级**：🔵 P3
