# 待办清单

> 根据当前代码实现情况整理的优先级任务。  
> 编号规则：P0（严重问题）> P1（高优先级）> P2（中优先级）> P3（低优先级）> P4（可选）。

---

## 🔴 P0 – 严重问题

| 编号 | 问题                                           | 说明                                                                                             | 解决方案                                                                                       |
| ---- | ---------------------------------------------- | ------------------------------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------- |
| #A1  | Grid 列/行定义导出丢失部分属性                  | 导入时存储了 `minWidth`/`maxWidth`，但导出时未正确处理 `Width="Auto"` 或 `*` 格式，可能输出错误。 | 在 `generateXAML` 中确保 `Width`/`Height` 属性原样输出，并保留 `MinWidth`、`MaxWidth` 等。      |
| #A2  | 事件绑定功能缺失（预览无实际响应）              | 编辑器内可设置 `EventType` 和 `EventData`，但预览时按钮点击无任何效果。                           | 在 `RenderManager` 中为按钮、列表项添加 `click` 监听，根据 `EventType` 执行 `window.open`/`alert` 等。 |
| #A3  | 导入 XAML 时组件 ID 可能冲突                    | `importFromXAML` 使用局部计数器生成 ID，未考虑与现有组件的全局唯一性。                            | 改为调用 `ComponentManager.getNextId()` 确保全局唯一。                                         |
| #A4  | 撤销/重做性能问题（全量快照）                  | 每次变动都深拷贝整个组件树，组件数量大时（>200）卡顿。                                           | 改为增量状态存储（操作日志）或限制快照频率（仅关键操作记录）。                                   |

## 🟠 P1 – 高优先级（功能缺陷或重大体验）

| 编号 | 任务                         | 说明                                                                 | 解决方案                                                                 |
| ---- | ---------------------------- | -------------------------------------------------------------------- | ------------------------------------------------------------------------ |
| #B1  | 快捷键全面支持               | 缺失 `Ctrl+S`（保存）、`Delete`（删除）、`Ctrl+D`（复制）。          | 在 `window` 键盘事件中增加对应分支，调用已有方法。                       |
| #B2  | 拖拽时滚动跟随               | 拖拽组件接近画布边缘时，预览区不自动滚动。                           | 在 `dragover` 中检测鼠标与容器边界的距离，动态调整 `scrollTop`。          |
| #B3  | 组件库拖拽预览图             | 从左侧拖拽组件时无跟随鼠标的占位图。                                 | 使用 `setDragImage` 自定义拖拽图像，显示组件图标和名称。                 |
| #B4  | 备份管理增强                 | 备份列表无搜索、批量删除；恢复前无差异对比。                         | 增加搜索框、多选删除、可选的“差异对比”弹窗。                             |
| #B5  | 超大组件树初次渲染优化       | 全量 `innerHTML` 重建，组件 > 200 时卡顿。                           | 引入虚拟滚动或增量 DOM 更新（保留节点，仅增删改移动）。                  |
| #B6  | 网格布局编辑器增强           | 不支持拖拽调整列/行顺序、复制定义。                                  | 增加排序按钮（上移/下移）和复制功能。                                     |
| #B7  | 属性面板实时响应防抖         | `change` 事件立即触发 `applyCurrentProps` 并刷新，高频输入导致重复渲染。 | 对 `applyCurrentProps` 增加 150ms 防抖。                                  |
| #B8  | 导入 XAML 时保留未知元素为注释 | 未知标签被直接忽略，用户可能丢失内容。                               | 将未知元素序列化为 XML 注释保留在组件树的特殊字段中，导出时还原。         |

## 🟡 P2 – 中优先级（体验优化）

| 编号 | 任务                                 | 说明                                                             | 解决方案                                                             |
| ---- | ------------------------------------ | ---------------------------------------------------------------- | -------------------------------------------------------------------- |

| #C4  | 移动端触屏拖拽                         | 手机/平板上无法拖拽组件。                                       | 使用 Touch 事件模拟 drag & drop，或增加“编辑模式”开关。             |
